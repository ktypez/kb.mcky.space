# LSN-001: แก้หน้าขาว + SW refresh loop บน Cloudflare Pages

# บทเรียน: หน้าขาว + Service Worker refresh loop

## อาการ
- เปิดหน้าเว็บแล้ว **หน้าขาว** (blank) หรือ
- หน้า **spam refresh เอง** วนลูปไม่หยุด

## รากเหตุ 2 อย่าง (ต่างกัน อย่าสับสน)

### 1. หน้าขาว — เกิดจาก build พัง ไม่ใช่ SW
- **สาเหตุจริง:** pnpm strict layout ทำให้ Rollup **externalize** dependency ที่เป็น bare import (`maplibre-gl`, `clsx`, `class-variance-authority`, `tailwind-merge`) → bundle มี bare specifier → runtime พัง → หน้าขาว
- ยิ่งถ้าไฟล์มีบรรทัด `'use client'` เป็นบรรทัดแรก (Next.js convention) Vite/Rollup จะยิ่งสับสน → externalize `maplibre-gl` ชัวร์
- **วิธีแก้:**
  - ลบ `'use client'` ออกจาก component ที่เป็น Vite SPA
  - ใน `vite.config.ts` เพิ่ม `resolve.alias` ชี้ทุก dep ไป path จริงใน `node_modules` + `resolve.dedupe` + `optimizeDeps.include`
  - เช็คหลัง build: `grep -rhoE "['\"](maplibre-gl|...)[ '\"]" dist/assets/*.js` ต้องไม่เจออะไรเลย

### 2. Refresh loop — เกิดจาก SW v1
- **สาเหตุ:** SW เวอร์ชั่นเก่า (`v1`) มี `triggerHeal()` ที่ postMessage `ASSET_STALE` แล้วทำ `location.reload(true)`
- เมื่อ Cloudflare SPA fallback คืน **HTML** สำหรับ asset path ที่หาไม่เจอ (hash เปลี่ยนหลัง deploy) SW เข้าใจผิดว่า chunk ตาย → reload → ได้ HTML → reload วนลูป
- **วิธีแก้:**
  - เขียน SW ใหม่ (`v2`): ทำแค่ cache app shell, asset ใช้ **network-first**, **ห้ามมี auto-reload**
  - `src/main.tsx`: ตอน load ให้ `navigator.serviceWorker.getRegistrations().then(r => r.forEach(x => x.unregister()))` **ก่อน** register SW ใหม่ (เคลียร์ SW v1 ที่ผู้ใช้ติดอยู่)
  - ตรวจสอบ live: `curl -s data.mcky.space/sw.js | grep VERSION` ต้องเป็น `v2`

## ขั้นตอนตรวจสอบ (verification)
รัน `node scripts/health-check.mjs` — ตรวจ 4 จุด:
1. หน้าแรก ERRORS none + URL คงที่ (ไม่ loop) + root มี DOM
2. `/maps` HTTP 200 (SPA fallback) + entry bundle ถูก serve
3. SW live = `v2` + ไม่มี `triggerHeal|ASSET_STALE`
4. bundle ไม่มี bare specifier

## หมายเหตุ
- WebGL context error ใน headless browser เกิดจาก **เครื่องไม่มี GPU** ไม่ใช่ bug ของแอป — ข้ามได้
- Git auto-deploy **ปิด** อยู่ → `npx wrangler pages deploy ./dist --project-name=data-mcky-space` คือผู้คุม deploy เดียว (ไม่ใช้ --branch)

## ไฟล์ที่เกี่ยวข้อง
- `vite.config.ts` — resolve.alias สำหรับ pnpm
- `public/sw.js` — SW v2 ปลอดภัย
- `src/main.tsx` — unregister SW เก่า
- `src/components/InlineMap.tsx` — ไม่มี 'use client', lazy-loaded
- `scripts/health-check.mjs` — ตรวจสภาพ
