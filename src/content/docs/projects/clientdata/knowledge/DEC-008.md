# DEC-008: Migrate to Astryx Design System

## Context

The app used a mix of shadcn/ui, Base UI, and hand-rolled CSS for its UI layer. This created inconsistency — form fields from one library, dialogs from another, icons from a third — with no unified design language. Styling relied on raw hex colors, making theming and dark mode brittle.

## Decision

Replace all UI dependencies with the Astryx Design System (v0.1.3, Meta) as the single source of truth for layout, spacing, and components. Use the Neutral theme preset. Replace `@phosphor-icons/react` with `lucide-react`.

## Alternatives Considered

- **Keep shadcn/ui + Base UI**: the hybrid approach had no unified design system, making dark mode maintenance painful
- **Switch to Tailwind UI**: paid, no React component primitives
- **Keep Phosphor Icons**: Lucide has wider Astryx compatibility

## Consequences

- Astryx handles all spacing, layout, and component styling via built-in props — no more `<div>` wrappers
- Theme tokens map from Astryx `--color-*` to Tailwind `@theme inline` in `globals.css`
- `data-astryx-theme="neutral"` on `<html>` activates the dark/light theme
- Removed: `@radix-ui/*`, `@base-ui/react`, `cmdk`, `@phosphor-icons/react`, `class-variance-authority`, `clsx`, `tailwind-merge`
- Astryx code spun off to separate repo `ktypez/astryx`, Vercel project `astryx` at `astryx.mcky.space`
- `ktypez/clientdata` master → `data.mcky.space` (old design)
