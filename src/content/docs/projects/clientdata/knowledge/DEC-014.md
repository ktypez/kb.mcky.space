# DEC-014: OKF MCP server wired into OpenCode

## Decision

Register the local OKF knowledge-base MCP server (`okf`) in OpenCode config at
`/home/clientdata/.config/opencode/opencode.jsonc` so the agent can read and
write the KB directly through tools instead of shelling out to read files.

## Config

```jsonc
"mcp": {
  "servers": {
    "okf": {
      "type": "local",
      "command": ["node", "/root/OKF/mcp-server/index.js"],
      "enabled": true
    }
  }
}
```

## How it works

The server is a stdio MCP server (`@modelcontextprotocol/sdk`) rooted at
`/root/OKF` (`lib/okf.js` resolves `OKF_ROOT` from its own location). Every tool
operates on the live markdown files in the KB â€” there is no separate database;
the filesystem IS the KB. Frontmatter (YAML between `---` fences) is the node
schema; the body after the fences is free markdown.

**Read tools**
- `okf_list_projects` â€” projects under `projects/` with file counts
- `okf_get_project` â€” concatenates `profile.md` + `agent.md` + `status.md`
- `okf_query_nodes` â€” filter nodes by `{project, type, status}` from frontmatter
- `okf_get_node` â€” single node by `{project, id}` (frontmatter + body)
- `okf_search` â€” case-insensitive full-text scan across all `.md` (skips `.git`,
  `node_modules`, `.next`), returns matching lines
- `okf_get_file` â€” raw content of any OKF-relative path
- `okf_list_dir` â€” directory tree of the KB
- `okf_doctor` â€” lifecycle audit (stale/expired/superseded), auto-expires
  nodes past their `expires` date

**Write tools**
- `okf_create_node` â€” auto-assigns ID prefix from `type`
  (`DEC`/`LSN`/`COMP`/`DOC`/`RSK`/`NODE`), bumps the highest existing number
- `okf_update_node` â€” patches frontmatter + optional body, stamps
  `last_updated`/`freshness`
- `okf_update_status` â€” shorthand to flip node `status`
- `okf_add_edge` â€” appends a typed `links:` entry (idempotent)

All write tools serialize back to `---<yaml>---\n\n<body>` on disk, so edits are
committed as real markdown the moment a tool runs.

## Rationale

- One shared MCP server gives every project agent consistent KB access
- Agents query nodes (e.g. `okf_query_nodes project:clientdata type:document
  status:active`) instead of grepping the filesystem
- Authoring/linking nodes is a tool call, keeping the graph edges valid

## Consequences

- `opencode.jsonc` and the MCP server path are external to the repo â€” changes
  require editor permission for `~/OKF/**` (already granted in config)
- If the server fails to boot, the agent falls back to reading KB markdown
  directly via shell/read tools
- `OKF_ROOT` is derived from the server's install location, so moving the
  `mcp-server` folder silently changes the KB root
