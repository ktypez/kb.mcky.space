# บทเรียน: ความคมชัดรูป + Layout collage

## 1) ปัญหารูปต้นฉบับเบลอ
- `roundImage()` ไม่มี `withoutEnlargement` -> รูปเล็กถูก upscale -> เบลอ
- แก้: `withoutEnlargement:true, fit:'fill', kernel:'lanczos3'` + รูปเล็ก center การ์ด + JPEG q90 mozjpeg

## 2) Layout: กลับเป็น MASONRY (max 3 คอลัมน์) + กันบั๊ก NaN
ผู้ใช้สั่งเปลี่ยน row-wrapping grid กลับเป็น masonry (เรียงลงคอลัมน์ที่สั้นที่สุด) เพราะรูปสั้น (152px) จะได้เติมช่องว่างแทนการขึ้นแถวใหม่คนเดียว
- Masonry: `colIdx = colHeights.indexOf(Math.min(...colHeights))` -> รูปสั้นเติมคอลัมน์ที่ว่าง -> collage สั้นลง (4 รูป tall/short: grid 648px -> masonry 424px)
- กันบั๊ก NaN: guard metadata `w=m.width>0?m.width:iw`, `h=m.height>0?m.height:iw`, `ih=round(iw*h/w)||iw` (ไอเดียผู้ใช้: วัดความสูงจากรูปแรกของแถว/คอลัมน์ + fallback iw)

## 3) ลบ isSmallLayout (ห้ามยุบคอลัมน์)
ผู้用户: "ใช่ ไม่ควรยุบ มีแต่จะเพิ่ม" -> ลบฟังก์ชัน isSmallLayout ทิ้ง ใช้ cols(count) เสมอ (max 3)
- ก่อนลบ: รูปแนวตั้ง >=ครึ่ง -> ยุบเหลือ 1 คอลัมน์ -> เรียงซ้อนยาว (เคส 4 รูป original ได้ 4004px)
- หลังลบ: กระจาย 3 คอลัมน์ตลอด -> เคสเดียวกันเหลือ 740px

## 4) บั๊ก "รูปกระจุกข้างบน" (เกิดตอนใช้ row-wrapping grid)
- คำนวณ y ก่อนสะสม gap -> ทับกัน -> แก้ด้วย rowTops สะสม (ปัจจุบันไม่เกี่ยวเพราะกลับเป็น masonry แล้ว)

## 5) Auto-cleanup R2: 90 -> 30 วัน
- r2-storage.js (default) + server.js (2 จุด)

## คำเตือนอนาคต
- ห้าม upscale รูปต้นฉบับ
- ทุกจุดคำนวณจาก metadata ต้อง guard NaN/0
- อย่ายุบคอลัมน์ตามสัดส่วนรูป (มีแต่จะยืด collage)
